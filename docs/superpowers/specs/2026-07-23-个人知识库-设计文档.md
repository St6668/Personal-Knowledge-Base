## 一、项目概述

一个面向个人使用的知识库管理系统，支持多格式知识上传、AI 驱动的智能问答、学习计划生成和面试式考察。

## 二、技术选型

| 层级 | 技术 | 说明 |
|------|------|------|
| 后端框架 | FastAPI (Python 3.11+) | 现代异步框架，性能优秀，自带 API 文档 |
| 数据库 | SQLite + SQLAlchemy | 存储结构化元数据 |
| 向量数据库 | ChromaDB | 轻量嵌入式向量存储，语义检索 |
| AI 生成 | DeepSeek Chat API | 负责理解、生成、评判（云端） |
| AI 嵌入 | BAAI/bge-small-zh-v1.5 | 本地 Embedding，中文语义检索（免费） |
| 前端 | Jinja2 服务端渲染 + 现代 CSS | MVP 阶段简单可维护，后续可升级前后端分离 |
| ASGI 服务器 | Uvicorn | FastAPI 推荐的生产级服务器 |

**职责划分：**
- **DeepSeek Chat API**：作为"生成大脑"——理解问题、生成答案、出题、评判回答
- **BGE Embedding（本地）**：作为"检索助手"——将文档向量化并存储，提问时找到最相关的文档片段喂给 DeepSeek
- **SQLite**：存储用户可见的结构化数据（文档元信息、计划、考察记录）
- **ChromaDB**：存储文本向量及其对应的原文片段，负责语义检索

## 四、功能模块设计

### 模块一：知识管理

| 功能 | 说明 |
|------|------|
| 文档上传 | 支持 PDF、Word (.docx)、TXT、Markdown、XMind (.xmind) |
| 文本笔记 | 在线编写 Markdown 笔记 |
| 知识库分组 | 可创建多个知识库（如"编程"、"历史"等）进行分类管理 |
| 标签系统 | 知识点可打标签，支持多标签筛选 |
| 文档列表 | 按知识库、标签筛选，支持搜索 |

**入库流程：**
```
上传文档 → 文件解析器（按类型分发）→ 提取纯文本 → 文本分块（chunk）
→ BGE Embedding 生成向量 → 向量 + 原文片段存入 ChromaDB
→ 文档元信息存入 SQLite
```

**解析器分发：**
- PDF → pdfplumber
- Word (.docx) → python-docx
- TXT → 直接读取
- Markdown → 保留结构信息提取文本
- XMind → xmindparser 提取主题结构和文本

**SQLite 核心表：**
- `knowledge_bases`：知识库分组
- `documents`：文档基本信息（标题、类型、知识库ID、来源、创建时间）
- `document_chunks`：文本分块（文档ID、序号、原文内容、向量ID）
- `tags`：标签表
- `document_tags`：文档-标签多对多关联

### 模块二：AI 对话

| 模式 | 说明 |
|------|------|
| 知识库问答 | 基于知识库回答，标注引用来源 |
| 自由对话 | 不检索知识库，直接与AI对话 |
| 范围锁定 | 可选择只在某个知识库/标签范围内检索 |
| 笔记整理 | 对话内容一键保存为新笔记 |

**对话流程：**
```
用户提问 → 问题向量化 → ChromaDB 语义检索 top-k → 构建上下文
（系统提示词 + 检索片段 + 对话历史）→ DeepSeek Chat Stream → 流式返回
→ 对话记录存入 SQLite
```

**SQLite 表：**
- `conversations`：对话会话（标题、模式、创建时间）
- `messages`：每条消息（角色 user/assistant、内容、引用文档ID列表、时间戳）

**前端交互：**
- 左侧对话列表，右侧聊天窗口
- AI 回复流式显示（打字机效果）
- AI 引用知识库内容时可点击跳转原文
- 新建对话、切换历史对话、删除对话

### 模块三：学习计划生成

**流程：**
```
用户选择知识范围（知识库/标签）→ 检索范围内所有知识点摘要
→ 构建提示词 → DeepSeek 生成结构化计划
→ 解析为模块列表 → 存入 SQLite → 前端展示
```

**SQLite 表：**
- `study_plans`：学习计划（标题、知识范围、模块总数、状态、创建时间）
- `plan_modules`：计划模块（标题、描述、知识点引用列表、建议时长、序号、状态）

**前端展示：**
- 纵向时间线/卡片列表
- 每个模块可展开查看详情和关联知识点
- 支持手动调整模块顺序
- 每个模块旁有"开始考察"按钮

### 模块四：AI 面试考察

**流程：**
```
选择模块/知识范围 → AI 生成第一道开放式问题 → 用户作答
→ AI 评判（准确性/完整性/深度）→ 追问 or 下一题
→ 循环 N 题后 → AI 给出本轮总结（评分/薄弱点/建议）
```

**设计要点：**
- **追问机制**：回答不完整时继续追问深挖
- **难度递进**：从基础到深入
- **开放性**：非选择题，检验真正理解
- **可中断**：随时结束，保留已完成记录

**SQLite 表：**
- `exam_sessions`：考察会话（关联计划模块、知识范围、题目数量、状态、时间、总分）
- `exam_questions`：每道题目（题目内容、用户回答、AI评判、评分、序号）

**考察结束后：**
- 生成考察报告（总分、逐题详情、薄弱点分析）
- 薄弱知识点可一键加入新学习计划

## 五、前端 UI 设计方向

- 风格：现代简洁，卡片式布局
- 配色：柔和中性色调，深色侧边栏 + 浅色内容区
- 核心页面：
  - 知识库首页（文档网格/列表）
  - AI 对话页（类聊天应用布局）
  - 学习计划页（时间线卡片）

## 八、MVP 不做的事情（明确边界）

- 不做用户认证/多用户（个人使用，单用户即可）
- 不做前后端分离（MVP 用 Jinja2 渲染）
- 不做图片 OCR 识别
- 不做网页链接自动抓取
- 不做定时任务/邮件提醒
- 不做移动端适配
- 不做自动化间隔重复算法（考察为手动触发）
