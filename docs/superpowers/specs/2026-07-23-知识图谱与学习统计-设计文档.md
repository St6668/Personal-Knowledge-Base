## 一、功能概述

在现有知识库系统基础上新增两个功能模块：

| 模块 | 路由 | 说明 |
|------|------|------|
| 知识图谱 | `/knowledge/graph` | AI 抽取概念节点 → ECharts 力导向图可视化 → 点击查看详情 |
| 学习仪表盘 | `/stats` | 从现有数据统计学习进度、答题趋势、薄弱点等，图表展示 |

## 二、知识图谱

### 2.1 数据模型

新增两张表（SQLite + SQLAlchemy ORM）：

**concepts — 概念节点**

| 字段 | 类型 | 说明 |
|------|------|------|
| id | Integer PK | 主键 |
| name | String(200) | 概念名，如"闭包" |
| description | Text | AI 生成的定义 |
| kb_id | Integer FK → knowledge_bases.id | 所属知识库 |
| source_chunks | Text | JSON: `[{doc_id, chunk_id, chunk_index}]` |
| importance | Float, default 0.5 | 1.0=核心概念, 0.0=边缘 |
| created_at | DateTime | 创建时间 |

**concept_relations — 概念关系边**

| 字段 | 类型 | 说明 |
|------|------|------|
| id | Integer PK | 主键 |
| source_concept_id | Integer FK → concepts.id | 源概念 |
| target_concept_id | Integer FK → concepts.id | 目标概念 |
| relation_type | String(50) | prerequisite_of / related_to / part_of |
| strength | Float | 0~1 关联强度 |
| description | Text | AI 对关系的说明 |

**唯一约束**：`(source_concept_id, target_concept_id, relation_type)`

### 2.2 概念抽取流程

```
用户选择知识库 → 点击"生成图谱"
  → 后端收集该知识库下所有文档的分块文本
  → 分批调用 DeepSeek API（每批最多 25 个分块）
  → 提示词：从文本中抽取概念 + 判断关系
  → 容错解析 JSON → 存入 concepts + concept_relations
  → 前端拉取图谱数据并渲染
```

**AI 输出格式**（每次请求）：

```json
{
  "concepts": [
    {"name": "闭包", "description": "函数可以记住并访问其词法作用域...", "importance": 0.8, "source_chunk_indices": [0, 3]},
    {"name": "作用域", "description": "变量可访问的范围...", "importance": 0.6, "source_chunk_indices": [0]}
  ],
  "relations": [
    {"source": "闭包", "target": "作用域", "type": "prerequisite_of", "strength": 0.9, "description": "理解闭包需要先掌握作用域"}
  ]
}
```

**去重策略**：同名概念按 kb_id 合并 source_chunks，importance 取最大值。

### 2.3 API 端点

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/knowledge/graph` | 图谱页面（HTML） |
| POST | `/knowledge/graph/generate` | 触发 AI 抽取概念，body: `{kb_id}` |
| GET | `/knowledge/graph/data?kb_id=` | 获取图谱 JSON（节点+边） |
| GET | `/knowledge/graph/concept/{id}` | 获取单个概念详情 |
| DELETE | `/knowledge/graph/reset?kb_id=` | 清除某知识库的图谱数据 |

### 2.4 前端渲染

- **库**：ECharts（CDN 加载 `echarts`），`echarts.init + force` 布局
- **节点大小** = 15 + importance × 25（px）
- **连线颜色**：prerequisite_of=朱砂红, related_to=灰色, part_of=翡翠绿
- **连线粗细** = strength × 4（px）
- **交互**：拖拽缩放（ECharts 内置）、点击节点弹出侧边详情面板、顶部搜索框高亮匹配节点、双击空白区域重置视图

### 2.5 侧边栏导航

新增导航项：
```
📊 知识图谱 → /knowledge/graph
```

## 三、学习统计仪表盘

### 3.1 数据来源

全部从现有表查询计算，**不新增存储表**：

| 指标 | 来源 | 计算方式 |
|------|------|---------|
| 知识库数 | knowledge_bases | COUNT |
| 文档数 | documents | COUNT |
| 概念数 | concepts | COUNT |
| 学习进度 | plan_modules | completed_count / total_count |
| 30天分数趋势 | exam_sessions | WHERE created_at >= date('now','-30 days') |
| 知识领域分布 | documents JOIN knowledge_bases | 按 kb_id 分组 COUNT |
| 薄弱点 TOP5 | exam_sessions.feedback | JSON 提取 weak_points，按频率排序 |
| 最近活动 | 各表 created_at | UNION ALL 按时间倒序，取最近 20 条 |

### 3.2 API 端点

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/stats` | 仪表盘页面（HTML） |
| GET | `/stats/overview` | 概览数据（卡片数字） |
| GET | `/stats/score-trend?days=30` | 30天分数趋势数据 |
| GET | `/stats/weak-points?limit=5` | 薄弱知识点 TOP N |
| GET | `/stats/activity?limit=20` | 最近活动时间线 |

### 3.3 页面布局

```
┌──────────────────────────────────────────────────┐
│  📊 学习仪表盘                                    │
├────────┬────────┬──────────┬────────────────────┤
│ 知识库  │  文档   │   概念    │     学习进度        │
│  3     │  25    │   47     │   12/20 (60%)      │
├────────┴────────┴──────────┴────────────────────┤
│                    │                             │
│  分数趋势 (折线图)    │  知识领域分布 (饼图)          │
│  30 天内            │  按知识库分组                │
│                    │                             │
├────────────────────┼─────────────────────────────┤
│  薄弱点 TOP5        │  最近活动                    │
│  1. 闭包    45分    │  · 07-23 完成 Python 考察    │
│  2. 原型链  50分    │  · 07-22 上传 3 篇文档       │
│  ...               │  · 07-21 生成知识图谱        │
└────────────────────┴─────────────────────────────┘
```

- 分数趋势用 ECharts **折线图**
- 领域分布用 ECharts **环形图**
- 薄弱点和活动时间线用纯 HTML/CSS 列表
