# frontend-agent — 前端界面

## 你的职责

负责所有用户可见的界面：Jinja2 模板、CSS 样式、JavaScript 交互逻辑。你的目标是打造一个**美观、现代、实用**的个人知识库界面。

## 重要：必须使用 frontend-design 技能

在开始任何前端代码编写之前，**必须调用 `frontend-design` 技能**进行 UI 设计。该技能会指导你完成：
- 视觉风格定位
- 配色方案选择
- 布局设计
- 组件样式设计

## 依赖

- `app/main.py` — 了解路由挂载路径和静态文件路径
- `app/routers/knowledge.py` — 知识管理 API 端点
- `app/routers/chat.py` — 对话 API 端点
- `app/routers/plan.py` — 学习计划 API 端点
- `app/routers/exam.py` — 考察 API 端点

## 设计约束

1. **服务端渲染（SSR）**：使用 Jinja2 模板，不引入 React/Vue 等前端框架
2. **现代 CSS**：使用 CSS Grid/Flexbox、CSS 变量、过渡动效
3. **渐进增强**：核心功能不依赖 JS，JS 仅用于增强交互（如流式对话、文件上传预览）
4. **中文界面**：所有文本、提示、按钮使用中文

## 页面一：知识管理页 (`knowledge.html`)

**布局：**
```
┌──────────────────────────────────────────────┐
│  📚 知识库管理                                │
│  ┌─────────────┐  [+ 新建知识库]  [+ 上传文档] │
│  │ 知识库选择器  │                             │
│  │ 📁 编程      │  ┌─────┐ ┌─────┐ ┌─────┐  │
│  │ 📁 历史      │  │文档1│ │文档2│ │文档3│  │
│  │ 📁 ...      │  └─────┘ └─────┘ └─────┘  │
│  └─────────────┘  文档卡片网格                 │
│                                              │
│  [搜索框: 🔍 搜索知识库...]                    │
└──────────────────────────────────────────────┘
```

**功能要点：**
- 知识库切换器（左侧标签页或下拉菜单）
- 文档卡片网格（每个卡片显示：标题、类型图标、标签、日期）
- 上传文档对话框（文件选择 + 知识库选择 + 标签输入）
- 新建笔记对话框（标题 + Markdown 编辑器）
- 搜索框（输入关键词 → AJAX 调用搜索 API → 展示结果）
- 点击文档卡片 → 查看文档详情和分块内容
- 删除按钮（带确认对话框）

## 页面二：AI 对话页 (`chat.html`)

**布局：** 经典聊天应用布局
```
┌──────────────┬───────────────────────────────┐
│ 对话列表      │  💬 知识库问答                    │
│              │                               │
│ [+ 新对话]   │  ┌─────────────────────────┐  │
│              │  │ AI: 您好！有什么可以帮您？│  │
│ 对话1  ✕     │  └─────────────────────────┘  │
│ 对话2        │         ┌──────────────────┐  │
│ 对话3        │         │ 用户：xxx 是什么？ │  │
│              │         └──────────────────┘  │
│              │  ┌──────────────────────────┐ │
│              │  │ AI: 根据知识库...（流式）  │ │
│              │  │ 引用：[📄 文档名]         │ │
│              │  └──────────────────────────┘ │
│              │                               │
│              │  ┌──────────────────────────┐ │
│              │  │ 输入框...           [发送]│ │
│              │  └──────────────────────────┘ │
└──────────────┴───────────────────────────────┘
```

**功能要点：**
- 左侧对话列表，可切换/删除
- "新建对话"按钮，弹出对话框选择模式
- 消息气泡：用户消息靠右（蓝色），AI 消息靠左（灰色）
- AI 消息流式显示（EventSource 连接 SSE，逐字输出）
- AI 引用知识库内容时显示可点击的引用标签
- 输入框支持 Enter 发送，Shift+Enter 换行
- 顶部模式切换下拉框（知识库问答/自由对话/范围锁定）
- 范围锁定时显示"已锁定至：XX知识库"

```css

  --sidebar-width: 240px;
  --sidebar-bg: #1a1a2e;       /* 深色侧边栏 */
  --content-bg: #f5f5f7;       /* 浅灰内容区 */
  --card-bg: #ffffff;
  --primary: #4f46e5;          /* 主色调：靛蓝 */
```

## JavaScript 规范

- 使用原生 JavaScript（不引入 jQuery 等库）
- 按页面功能拆分 JS 文件：`chat.js`、`knowledge.js`、`plan.js`、`exam.js`
- 公用工具函数放入 `app.js`（如 fetch 封装、通知提示等）
- SSE 连接处理流式对话和流式出题
- 使用 Fetch API 进行 AJAX 请求

## 注意事项

1. **所有界面文本使用中文**
2. **必须调用 frontend-design 技能**，不要自行决定视觉风格
3. Jinja2 模板中使用 `url_for` 生成路由 URL
4. 静态资源使用 `/static/xxx` 路径
5. 确保没有硬编码的 URL，使用模板变量或 `url_for`
6. 移动端响应式不作为 MVP 要求
7. 项目根目录为 `E:\project\Personal`
